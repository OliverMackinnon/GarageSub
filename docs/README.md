# GarageSub Documentation

Welcome to the documentation hub for the GarageSub project.

## How this Documentation is Organized

### 1. [Tutorials](./Tutorials)
**"I am learning."**
* **Goal:** Learning-oriented.
* **Content:** Lessons that take the reader by the hand through a series of steps to complete a project.
* **Example:** *“ assembling your first GarageSub frame”* or *“Your first pool test.”*

### 2. [How-To Guides](./How-To-Guides)
**"I am trying to solve a specific problem."**
* **Goal:** Problem-oriented.
* **Content:** Directions that guide the reader through a specific real-world task or problem. These are like recipes.
* **Example:** *“Calculating Buoyancy”*, *“Calibrating the IMU”*, or *“Replacing a thruster.”*

### 3. [Reference](./References)
**"I need information."**
* **Goal:** Information-oriented.
* **Content:** Technical descriptions of the machinery and code. Accurate, structured data.
* **Example:** *Wiring diagrams, API endpoints, Screw sizes, or Material datasheets.*

### 4. [Explanation](./Explanation)
**"I want to understand."**
* **Goal:** Understanding-oriented.
* **Content:** Discursive, theoretical material. It explains *why* things are the way they are.
* **Example:** *“Why we chose aluminum extrusions over PVC”* or *“The physics of underwater pressure.”*

### Contributing
If you would like to contribute to the GarageSub documentation, please determine which quadrant your document belongs to and submit a Pull Request to the corresponding folder.
